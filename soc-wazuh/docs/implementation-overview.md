# Implementation Overview

## Overview

This project demonstrates the deployment and configuration of Wazuh SIEM for centralized security monitoring in an enterprise-style environment.

The implementation focused on deploying the Wazuh Manager, integrating Windows and Linux agents, enabling File Integrity Monitoring, collecting application and web server logs, integrating Office 365 audit monitoring, and designing custom dashboards for security visibility.

---

## Implementation Scope

The implementation included:

- Wazuh Manager deployment on Ubuntu Server
- Virtualized deployment using VMware ESXi
- Windows endpoint agent deployment
- Windows Server agent deployment
- Linux web and application server integration
- Group Policy Object (GPO) based agent deployment
- File Integrity Monitoring configuration
- Office 365 audit log monitoring
- Custom dashboard planning and visualization

---

## Key Security Objectives

- Centralize security monitoring across endpoints and servers
- Improve visibility into authentication, system, and application activity
- Monitor file changes on sensitive servers
- Collect Office 365 audit events
- Support threat detection and investigation workflows
- Build dashboards for SOC-style monitoring

---

## Implementation Notes

Sensitive details such as real IP addresses, credentials, organizational names, tenant identifiers, and internal infrastructure information have been removed or generalized.

The purpose of this documentation is to present the technical implementation approach while protecting the confidentiality of the original environment.

---

## Conclusion

The implementation provided practical experience in enterprise SIEM deployment, endpoint monitoring, cloud audit integration, File Integrity Monitoring, and dashboard-based security visibility using Wazuh.
