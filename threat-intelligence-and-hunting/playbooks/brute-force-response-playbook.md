# Brute-Force Attack Response Playbook

## Objective
Detect, contain, and respond to brute-force authentication attacks targeting servers, endpoints, or network services.

## Scope
This playbook applies to repeated unauthorized login attempts against services such as SSH, RDP, VPNs, web portals, or administrative interfaces.

## Incident Categories
- SSH brute-force attacks
- RDP brute-force attempts
- Password spraying attacks
- Credential stuffing attempts
- Repeated failed authentication events

## Detection Sources
- Wazuh SIEM alerts
- Authentication failure logs
- Syslog and SSH logs
- Active Response alerts
- Firewall logs
- Threat intelligence indicators

## Response Procedure

### 1. Detection
- Review authentication failure alerts generated by Wazuh.
- Identify targeted systems, usernames, source IP addresses, and attack frequency.
- Determine whether the attack is isolated or distributed across multiple sources.

### 2. Analysis
- Review SSH, authentication, and firewall logs.
- Check whether any login attempts were successful.
- Correlate events with threat intelligence indicators where applicable.
- Investigate whether the activity matches known brute-force behavior patterns.

### 3. Containment
- Block malicious IP addresses using firewall rules or Wazuh Active Response.
- Enable temporary account lockout if needed.
- Restrict exposed administrative services where possible.
- Isolate affected systems if compromise is suspected.

### 4. Eradication
- Remove unauthorized access or persistence mechanisms if compromise occurred.
- Reset affected credentials.
- Harden authentication policies and disable unused services.

### 5. Recovery
- Restore normal access after validation.
- Monitor systems for repeated attack attempts.
- Verify that detection and blocking mechanisms are functioning correctly.

### 6. Post-Incident Review
- Document attack timeline, targeted systems, and response actions.
- Review firewall and SIEM detection effectiveness.
- Improve authentication controls and monitoring rules.

## Roles and Responsibilities
- SOC Analyst: Alert monitoring and investigation.
- Security Engineer: Firewall blocking and detection tuning.
- System Administrator: Service hardening and credential management.
- Incident Response Lead: Coordination and reporting.

## Validation
- Perform controlled brute-force simulations in a lab environment.
- Validate Active Response and firewall-drop automation functionality.
- Review and update the playbook periodically.
